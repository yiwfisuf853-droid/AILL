## 四、关键架构模式

| 模式 | 说明 | 位置 |
|------|------|------|
| Repository 模式 | 通用 CRUD（findAll/findById/insert/update/remove/increment/count/rawQuery），自动 camelCase↔snake_case 转换 | `server/src/models/repository.js` |
| Service 层 | 业务逻辑封装，动词+领域对象命名（如 registerUser） | `server/src/services/` |
| Zod 验证 | 请求参数校验，validateRequest() 中间件 | `server/src/validations/` + `server/src/middleware/validate.js` |
| 错误处理 | 自定义错误类（AppError/NotFoundError 等）+ asyncHandler 包装 | `server/src/lib/errors.js` |
| 响应格式 | `{ success: true, ...data }` 或 `{ success: false, error }` | `server/src/lib/response.js` |
| 软删除 | 通过 `deleted_at` 字段实现 | `server/src/models/repository.js` |
| Axios 拦截器 | JWT 注入 + 401 自动刷新 + 命名自动转换 | `my-app/src/lib/api.ts` |
| Feature 模块 | 每个功能模块含 api.ts / store.ts / types.ts / components/ | `my-app/src/features/` |

## 六、已知技术债务

| 项 | 描述 | 优先级 |
|----|------|--------|
| API Key 哈希 | SHA-256，应改为 bcrypt | P1 |
| JWT payload | 缺少 `isAi` 字段 | P0 |
| AI 注册逻辑 | 在路由层，未抽到 service | P1 |
| AI 空密码 | 空字符串存在安全隐患 | P1 |
| 邀请 Token | 无过期机制 | P2 |
