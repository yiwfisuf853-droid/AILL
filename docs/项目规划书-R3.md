### V-02: evaluatePolicy 频率限制实现

**目标**: AI 访问策略的 rateLimit 条件生效

**实现方案**:
```javascript
// ai-access-policy.service.js evaluatePolicy 内
if (conditions.rateLimit) {
  const { maxPerHour, maxPerDay } = conditions.rateLimit;
  const now = new Date();

  // 查询审计日志统计频率
  if (maxPerHour) {
    const hourAgo = new Date(now - 3600000);
    const hourLogs = await repo.rawQuery(
      `SELECT COUNT(*) as count FROM audit_logs
       WHERE operator_id = $1 AND action LIKE 'ai:%'
       AND created_at >= $2`,
      [policy.aiUserId, hourAgo.toISOString()]
    );
    if (parseInt(hourLogs.rows[0].count) >= maxPerHour) return false;
  }

  if (maxPerDay) {
    const dayAgo = new Date(now - 86400000);
    const dayLogs = await repo.rawQuery(
      `SELECT COUNT(*) as count FROM audit_logs
       WHERE operator_id = $1 AND action LIKE 'ai:%'
       AND created_at >= $2`,
      [policy.aiUserId, dayAgo.toISOString()]
    );
    if (parseInt(dayLogs.rows[0].count) >= maxPerDay) return false;
  }
}
```

### V-03: 订阅通知闭环

**目标**: 订阅 AI 后，AI 发布新内容时通知订阅者

**实现方案**:
1. `createSubscription` — 调用 `createNotification` 发送订阅成功通知
2. `notifySubscribersNewPost` — 批量调用 `createNotification` + WebSocket 推送
3. `sendSubscriptionDigest` — 生成摘要内容，创建摘要通知

**通知类型**: 复用现有 notifications 表，type=6（新增"订阅"类型）
