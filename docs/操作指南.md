## 一、快速启动

### 环境要求

- Node.js 18+
- PostgreSQL 16+
- npm 9+

### 后端启动

```bash
cd server
cp .env.example .env    # 首次配置环境变量
npm install
npm run dev              # http://localhost:3000
```

### 前端启动

```bash
cd my-app
npm install
npm run dev              # http://localhost:5173（代理 /api → :3000）
```

### Docker 启动

```bash
docker compose up -d --build
```

### 首次运行

```bash
cd server
npm run init-db          # 初始化数据库 Schema（37 张表）
npm run seed             # 填充测试数据
```

### 数据库迁移

```bash
cd server
npm run migrate          # 运行列迁移
```

## 二、环境变量

### 后端（server/.env）

| 变量 | 说明 | 必须 | 默认值 |
|------|------|------|--------|
| `PG_HOST` | PostgreSQL 主机 | 是 | — |
| `PG_PORT` | PostgreSQL 端口 | 是 | — |
| `PG_DATABASE` | 数据库名 | 是 | — |
| `PG_USER` | 数据库用户 | 是 | — |
| `PG_PASSWORD` | 数据库密码 | 是 | — |
| `JWT_SECRET` | JWT 签名密钥 | 是 | — |
| `JWT_REFRESH_SECRET` | Refresh Token 密钥 | 是 | — |
| `PORT` | 后端端口 | 否 | 3000 |

> `JWT_SECRET` 和 `PG_PASSWORD` 缺失时服务拒绝启动。

### 前端

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `VITE_API_URL` | 后端 API 地址 | `http://localhost:3000` |

## 五、核心交互

### 搜索
- TopBar 搜索框输入关键词回车 → 跳转 `/search?q=xxx`
- 快捷键 `⌘K` / `Ctrl+K` 聚焦搜索框
- 搜索结果标题和摘要高亮匹配词
- 支持分区过滤 + 排序（相关/最新/最热）

### 评论区
- 嵌套回复：顶级评论 → 「查看 N 条回复」展开子评论
- 最多 3 层嵌套，缩进 + 竖线视觉层级
- 点击回复自动预填 `@username`
- 排序切换：最新 / 最热

### 通知
- 7 种类型 Tab：全部 / 点赞 / 评论 / 关注 / 系统 / 活动 / 订阅
- WebSocket 实时推送
- 标准分页（每页 20 条）

### 左侧栏
- 展开/收起切换（桌面端底部按钮）
- 收起：48px 半透明轨道，hover 展开为 overlay
- 分区快捷入口 + 个人区（通知/私信/收藏/我的/设置）
- 移动端：抽屉式

### 右侧栏
- 用户卡片（头像+用户名+快捷操作）
- 快捷操作（发帖子 + AI 中心）
- 热门讨论（Top 5）
- 热门标签

### 顶栏
- 默认 48px，滚动后扩展至 56px
- 滚动后显示面包屑导航 + 搜索加宽
- 模块色彩边框

### 底栏
- 位于内容区底部（所有页面可滚动到底部看到）
- 品牌 + 快速导航 + 关于

## 七、开发规范

### 新增功能流程

```
后端: routes/ → validations/(Zod) → services/ → models/repository.js
前端: api.ts → store.ts → types.ts → components/
```

### 命名约定

| 维度 | 规范 | 示例 |
|------|------|------|
| 后端 JS | camelCase | `getUserById` |
| PostgreSQL | snake_case | `created_at` |
| 前端 TS/TSX | camelCase | `handleSubmit` |
| data-name | 分层驼峰 | `homeHeroTitle` |
| CSS class | camelCase | `layoutShell` |

> Repository 和 Axios 拦截器自动转换 camelCase↔snake_case，不手动处理字段名。

### 认证方式

```
JWT:      Authorization: Bearer <jwt_token>
API Key:  Authorization: Bearer aill_xxxxx（R6 新增）
```

认证优先级：JWT > API Key > 未认证

## 九、故障处理

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 后端启动失败 | 缺少环境变量 | 检查 `server/.env` 中 `PG_*` 和 `JWT_*` |
| 数据库连接失败 | PostgreSQL 未运行或密码错误 | 确认 PG 服务运行，检查 `PG_PASSWORD` |
| 前端代理 404 | 后端未启动 | 先启动后端 `npm run dev` |
| WebSocket 断连 | Token 过期 | 前端自动刷新，检查 refresh token |
| 帖子列表为空 | 未执行 seed | `cd server && npm run seed` |
| 迁移报错 | 表结构版本不一致 | `npm run init-db` 重建（开发环境） |
